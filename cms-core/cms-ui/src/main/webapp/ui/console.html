        • Query entities

        Parameters

        Parameter Value Description
        repo repository name
        branch branch name
        query query expression to be executed

        Options

        Parameter Value Description
        mode uri:query result as href style
        normal:query result without href
        explain the query result has explanation or not
        sortOn sort the query result based on field name
        sortOrder sort the query result in an ascending order or a descending order
        limit limit the query result
        maxFetch the max fetch size that would control the sub-query data fetch size when query database. This value takes effect only when the query have sub-query.
        paginationMode The pagination mode that to be used. Default is SKIP_BASED.
        skip skip the query result. Applied only when paginationMode=SKIP_BASED.
        cursor the cursor value for the pagination purpose. Applied only when paginationMode=ID_BASED.
        hint hint the query result
        allowFullTableScan allow full table scan or not
        X-CMS-PRIORITY HTTP Headers
        X-CMS-CONSISTENCY HTTP Headers
        Authorization to authenticate service call

      • get /audit/{repo}/{branch}/{metatype}/{oid}

        • Query history - Only Supports Production CMS Audit

        Parameters

        Parameter Value Description
        repo repository name
        branch branch name
        metatype metadata name
        oid object id
        startTime start of the time range, default 7 days before today. Format: Unix Timestamp or Date(YYYYMMDD e.g.20131225). Please adjust it to your needs!
        endTime end of the time range, default now. Format: Unix Timestamp or Date(YYYYMMDD e.g.20131228). Please adjust it to your needs!

        Options

        Parameter Value Description
        uid user id
        cid component id, which is stored as _modifier in the entity
        fieldName field name to be queried
        delta query delta changes or not. If set 'true', you should specify 'fieldName' parameter value as well
        limit maximum records returned, default 100
        sortOrder sort the query result by timestamp in an ascending order or a descending order
        Hide Response

        Response Code

        Response Body
